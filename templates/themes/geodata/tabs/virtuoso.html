Virtuoso SPARQL endpoint

We provide direct access to a SPARQL endpoint from our Virtuoso server version 7.1.

Through this standard Virtuoso interface, you can submit your SPARQL queries using the native Virtuoso SPARQL protocol endpoint and you may access both data and metadata from http://geodata.gov.gr that are INSPIRE-compliant. The only difference is the Default Data Set Name (graph IRI) that you must specify in your SELECT of CONSTRUCT queries against this native interface:
If you want to query:

  • INSPIRE metadata: specify urn:x-geoknow-eu:sparql:virtuoso:metadata:inspireas your Graph IRI.
  • INSPIRE data: specify urn:x-geoknow-eu:sparql:virtuoso:data:inspireas your Graph IRI.
Note that you cannot submit any queries against INSPIRE Catalogue Services (CSWs) through this native interface.

Please consult the Virtuoso Wiki page describing the service or section RDF Database and SPARQL in the Online Virtuoso Documentation .

Of course, through our rich, customized Web-based interface at http://geodata.gov.gr/sparql/, you may still submit any SPARQL queries (and not only the ones available as examples). All interfaces interact with the same Virtuoso server version 07.10.3207 on Linux (x86_64-bit-linux-gnu), Single Server Edition by OpenLink Software.

For your convenience we have a set of predefined name space prefixes and inference rules, which can be used in SPARQL queries.

A note on GeoSPARQL support in Virtuoso

GeoSPARQL is a recent (2012) OGC standard that suggests a unified approach for representing linked geospatial data as RDF triples and querying them through a SPARQL extension equipped with a variety of spatial operators and functions. Virtuoso is very robust and highly scalable, and has recently (version 7.1, spring 2014) included support for storing several spatial data types (points, linestring, polygons, etc.) in the RDF repository. However, it is not GeoSPARQL-compliant yet and has limited support for geometric processing. For example, testing containment of a point in a polygon is evaluated against the minimum bounding box of the polygon and not against the exact geometry. This functionality will be extended in future releases of the software.

Please consult OpenLink Virtuoso manuals on RDF and Geometry, as well as the latest upgrades regarding geometric support.
